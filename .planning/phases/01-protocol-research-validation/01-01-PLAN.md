2. **Required Models** — Table listing each SunSpec model dbus-fronius expects:
   - Model 1 (Common): MANDATORY. C_Manufacturer must be "Fronius" (padded with nulls to 32 bytes). C_Model, C_Version, C_SerialNumber passed through. Length = 65 registers.
   - Model 103 (Three-Phase Inverter): MANDATORY. Provides AC/DC measurements, power, energy, status. Length = 50 registers.
   - Model 120 (Nameplate): MANDATORY for operation. Provides WRtg (rated power). Length = 26 registers.
   - Model 123 (Immediate Controls): OPTIONAL but needed for power limiting. Provides WMaxLimPct write target. Length = 24 registers.
   - End Marker: 0xFFFF followed by 0x0000.

3. **Unit ID** — dbus-fronius defaults to unit ID 126 for SunSpec devices. Proxy must respond on unit ID 126.

4. **Fronius-Specific Behaviors** — Power limiting auto-enabled, Solar API fallback (not needed for SunSpec path), null-frame filter (Status=7 with all-zero data discarded), split-phase handling.

5. **Protocol Details** — Integer encoding with scale factors (not float). Scale factor = signed int16, actual_value = raw_value * 10^sf. Big-endian byte order.

6. **What the Proxy Must Emulate** — Checklist of 6 items: (1) "SunS" at 40000, (2) C_Manufacturer="Fronius", (3) model chain Common->103->120->123->End, (4) unit ID 126, (5) integer+SF encoding, (6) no HTTP dependency.
