Create `scripts/validate_se30k.py` — a comprehensive Modbus TCP validation script that connects to the SolarEdge SE30K at 192.168.3.18:1502 (unit ID 1) and validates the register layout documented in the research and mapping spec.

The script must perform these checks IN ORDER and print results clearly:

**1. Connection Test**
- Connect to 192.168.3.18 port 1502 via ModbusTcpClient
- Print connection status
- If connection fails, print error and exit with code 1

**2. SunSpec Header Validation**
- Read 2 registers at address 40000 (slave=1)
- Pack as big-endian bytes and verify ASCII "SunS" (0x53756E53)
- Print: `SunSpec Header: {bytes} — {"PASS" or "FAIL"}`

**3. Common Model (Model 1) Validation**
- Read 67 registers at address 40002 (2 header + 65 data)
- Verify DID = 1, Length = 65
- Extract and print C_Manufacturer (registers 2-17, 16 regs = 32 bytes, decode as ASCII, strip nulls)
- Extract and print C_Model (registers 18-33)
- Extract and print C_Version (registers 42-49, 8 regs = 16 bytes)
- Extract and print C_SerialNumber (registers 50-65, 16 regs = 32 bytes)
- Extract and print C_DeviceAddress (register 66)
- Print: `Common Model: DID={did}, Length={length} — {"PASS" or "FAIL"}`

**4. Inverter Model Validation**
- Read 2 registers at address 40069 to get header (DID, Length)
- Print: `Inverter Model: DID={did}, Length={length}`
- Verify DID is 101, 102, or 103 (any single/split/three-phase)
- Read the full model data (length registers at 40071)
- Extract and print key values using struct.unpack for signed int16:
  - I_AC_Current (40071) with SF (40075): `AC Current: {raw} * 10^{sf} = {actual} A`
  - I_AC_Power (40083) with SF (40084): `AC Power: {raw} * 10^{sf} = {actual} W`
  - I_AC_Frequency (40085) with SF (40086): `Frequency: {raw} * 10^{sf} = {actual} Hz`
  - I_AC_Energy_WH (40093-40094, acc32) with SF (40095): `Lifetime Energy: {actual} Wh`
  - I_DC_Voltage (40098) with SF (40099): `DC Voltage: {actual} V`
  - I_DC_Power (40100) with SF (40101): `DC Power: {actual} W`
  - I_Status (40107): `Status: {value} ({name})`
  - I_Status_Vendor (40108): `Vendor Status: {value}`

**5. Model Chain Walk**
- Starting after the inverter model (40069 + 2 + inverter_length), walk the model chain:
  - Read 2 registers (DID, Length) at current address
  - Print: `Model at {addr}: DID={did}, Length={length}`
  - If DID == 0xFFFF, print "End of model chain" and stop
  - Otherwise advance by (2 + length) and continue
  - Collect all found model DIDs into a list
- Print: `Model chain: {list_of_dids} -> END`
- This answers the open question: does SE30K have Model 120 or 123?

**6. Summary**
- Print a summary table with all PASS/FAIL results
- Print whether Model 120 exists (if not: "Proxy must SYNTHESIZE Model 120")
- Print whether Model 123 exists (if not: "Proxy must SYNTHESIZE Model 123 and translate writes to 0xF300/0xF322")
- Exit with code 0 if all critical checks pass, 1 if any fail

Helper function for reading string registers:
```python
def regs_to_string(registers):
    """Convert register list to ASCII string, stripping null bytes."""
    raw = b''.join(r.to_bytes(2, 'big') for r in registers)
    return raw.decode('ascii', errors='replace').rstrip('\x00').strip()
```

Helper function for signed int16:
```python
def to_int16(value):
    """Convert unsigned uint16 to signed int16."""
    return struct.unpack('>h', struct.pack('>H', value))[0]
```
